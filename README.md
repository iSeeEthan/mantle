> [!WARNING]

> [!IMPORTANT]
> Mantle is designed around realism-first world generation rather than vanilla gameplay balance.

## Overview

Mantle replaces Minecraft's terrain with a world shaped by a plate-tectonic simulation.
Instead of layering noise to fake the look of mountains and valleys, Mantle simulates the
processes that actually produce them, like crustal uplift, fluvial erosion, drainage, and rock
weathering. The result is a landscape that holds together: rivers run downhill into the sea,
valleys sit where water carved them, mountain ranges rise where the crust was pushed up, and
hard rock stands out as ridges while soft rock wears into broad valleys.

The simulation runs once per seed and is fully deterministic, so the same seed always produces
the same terrain down to the block.

## Features

- **Tectonic terrain** shaped by plate uplift, fluvial erosion, and drainage rather than noise.
- **Rock-controlled erosion.** A layered rock column controls how fast the surface wears down, so
  hard rock forms escarpments and ridges while soft rock erodes into valleys.
- **Rivers and lakes** traced from headwaters to the ocean, with no floating water or stone sills.
- **Karst.** Soluble rock dissolves into dry sinkholes and feeds larger cave systems below.
- **Caves.** Winding tunnels and occasional caverns carved underground, denser in soluble rock.
- **Sediment deposition** that lays eroded material back down in low-gradient reaches, building
  floodplains and deltas.
- **Coastal spawn.** Players always start on dry land near the sea.
- **`/mantle` commands** for finding and teleporting to terrain features (see below).

## Commands

| Command | What it does |
| --- | --- |
| `/mantle locate <target>` | Finds the nearest matching feature and prints clickable coordinates that fill a teleport command. |
| `/mantle tp <x> <z>` | Teleports to coordinates, snapping to the surface. |
| `/mantle tp spawn` | Teleports to world spawn. |
| `/mantle tp coast` | Teleports to the nearest coast. |

`locate` targets include `river`, `lake`, `river_mouth`, `coast`, `ocean`, `peak`, `deep_ocean`,
`plains`, `cliff`, `cave`, `sinkhole`, and the four rock types (`sedimentary`, `igneous`,
`metamorphic`, `basement`).

## How it works

Terrain is generated by a custom NeoForge chunk generator backed by an offline simulation grid
(1280 by 1280 cells over a 10,000-block world, roughly 7.8 blocks per cell). Hydrology runs on a
2x finer grid so rivers resolve to a few blocks per cell.

- **Tectonic uplift.** A plate field defines where the crust rises, seeding the initial
  heightfield.
- **Stream-power erosion.** A deterministic landscape-evolution solver carves the uplifted crust
  using the standard fluvial incision law. Erosion and drainage are re-solved together each step,
  so channels migrate and capture one another, producing a coherent dendritic river network
  rather than noisy gullies. Erosion strength varies with the underlying rock, so geology shows
  through in the shape of the land.
- **Sediment transport.** Material eroded upstream is routed downhill and deposited where the
  gradient flattens, smoothing valley floors and building up river mouths.
- **Drainage.** Priority-Flood depression filling, D8 flow directions, and flow accumulation
  determine where water collects and how large each river becomes.
- **River rasterization.** The river network is stamped at block resolution with flat per-cell
  water surfaces, so rivers stay connected from headwaters to the ocean without floating water or
  stone sills.
- **Stratigraphy.** A deformed stack of rock layers is sampled per column, driving erosion
  resistance, karst, and where each rock type reaches the surface.
- **Karst and caves.** Soluble rock near the surface dissolves into dry sinkholes, and a 3D noise
  field carves tunnels and caverns underground, with denser caves in soluble rock.

While a world is being created, the loading screen shows the current simulation stage and, once
the world is built, chunk-generation progress.

## Building

Requires JDK 21.

```bash
./gradlew build
```

The built mod jar lands in `build/libs/`.

To run the development client:

```bash
./gradlew runClient
```

> [!NOTE]
> The generator is baked into a save when it is first created, so Mantle terrain only appears in
> a freshly created world, not existing saves.

## Project layout

| Path | Contents |
| --- | --- |
| `src/main/java/.../world/gen/tectonics/` | The simulation: tectonics, erosion, drainage, rivers, strata, karst, caves |
| `src/main/java/.../world/gen/` | The Minecraft chunk generator |
| `src/main/java/.../world/` | World access, feature locator, generation status |
| `src/main/java/.../world/noise/` | Deterministic value and gradient noise |
| `src/main/java/.../command/` | The `/mantle` command |
| `src/main/java/.../client/` | Client-side loading screen overlay |
| `verify/` | Standalone offline tools that render and validate the terrain without launching Minecraft |
